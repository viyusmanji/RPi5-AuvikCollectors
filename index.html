Raspberry Pi 5 + Auvik Collector
Deployment Guide

Standard operating procedure for deploying Raspberry Pi 5 devices with PoE+ HATs as dedicated Auvik network monitoring collectors at client sites. A single Ethernet cable delivers both power and data — no additional infrastructure required.

Version 1.0
Updated 2026-02-03
Platform ARM64 / Bookworm
Status Active
About This Guide
This is a public technician reference document. It covers hardware selection, OS configuration, Auvik collector installation, networking requirements, and field deployment procedures for the Pi 5 + PoE + Auvik stack.

Optional Add-ons

  • NVMe SSD + M.2 HAT: For sites with SD card reliability concerns. Some combo PoE+NVMe HATs exist (GeeekPi P33) that stack both — evaluate case compatibility.
  • USB-C 27W Power Supply: Keep one per tech bag as a PoE fallback if client switch doesn't support 802.3at or has insufficient power budget.

PoE Switch Compatibility

Critical
Verify the client switch supports 802.3at (PoE+) on the target port. Most Pi 5 PoE HATs will not negotiate on 802.3af-only ports. Check the switch power budget — especially on switches with many PoE devices already consuming wattage.

04 Auvik Collector Installation

Auvik supports two officially documented methods for ARM64 / Raspberry Pi deployments. For dedicated single-purpose Pis (our standard), use the Bash Script Install.

Recommended  Native Bash Script Install

The collector takes over the entire machine — no other workloads can run. This is the intended model for a dedicated Pi collector appliance.

Alternative  Docker Container

Use only if the Pi will run additional workloads alongside Auvik. The ARM64 Docker image is available on Docker Hub (latest tag). The collector auto-detects ARM64 — no emulation needed.

Bash Script Installation Steps

  1. Flash Raspberry Pi OS Lite (64-bit) to the microSD card using Raspberry Pi Imager. Pre-configure hostname, SSH key, and locale via the imager's advanced settings (Ctrl+Shift+X).
  2. Insert the microSD, connect Ethernet to a PoE+ switch port, and allow the device to boot. Confirm power via PoE (no USB-C cable needed).
  3. SSH into the Pi: ssh viyuadmin@{ip-address}. Verify network connectivity: ping 8.8.8.8 and curl -I https://auvik.com.
  4. Log into Auvik → navigate to the target client/site → click Add Collector → select Bash Script.
  5. Copy the generated install command (it contains the site-specific token). Run it via SSH on the Pi.
  6. The script installs the collector agent and reboots the device. After reboot, the collector should appear online in the Auvik dashboard within 5–10 minutes.
  7. Verify device discovery is populating — switches, APs, firewalls should begin appearing on the network map.
Pi OS Shortcut
Per Auvik docs: if the Pi is already running a supported Ubuntu OS or Raspberry Pi OS, you can skip straight to step 4 and run the installation script directly.

05 Network Configuration

IP Addressing

Each Pi requires a static IP or DHCP reservation. Static is preferred. Record the IP in your documentation platform under the client site record.

VLAN Placement

Place the Pi on the management VLAN with inter-VLAN routing to all monitored subnets. The Auvik collector uses SNMP, WMI, and other discovery protocols that require Layer 3 reachability. If the client uses strict segmentation, configure ACLs to allow the Pi's IP to reach target devices.

Firewall / Outbound Access

The collector requires outbound HTTPS (TCP 443) to Auvik's cloud. No inbound ports from the internet are needed. Verify the client firewall does not block or SSL-inspect traffic from the collector's IP. If a web proxy exists, configure the collector accordingly per Auvik's proxy documentation.

SNMP Configuration

Enable SNMP on all managed infrastructure — switches, routers, firewalls, access points. Configure SNMPv2c (community string) or preferably SNMPv3 (auth + encryption). Add the Pi's static IP to each device's allowed SNMP hosts list. Enter credentials in Auvik under the client site's credentials section.

06 Remote Access

Since the Auvik bash script takes over the Pi, traditional RMM agents may not be installable after deployment. Set up remote access before running the Auvik install.

Option A — Tailscale (Recommended)

Install Tailscale before deploying the Auvik collector. It provides a persistent mesh VPN tunnel back to the management network with zero-config NAT traversal. Every deployed Pi appears in the Tailscale admin console for SSH access and health monitoring.

curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --ssh

Option B — ZeroTier

Alternative mesh VPN if Tailscale is not viable. Same concept — persistent overlay network for remote SSH.

Option C — Reverse SSH Tunnel

If no mesh VPN is feasible, configure a persistent reverse SSH tunnel via autossh or a systemd service to a viyu.net jump host.

Auvik Offline Alerts
Auvik automatically alerts when a collector goes offline. Configure notifications to your NOC/operations channel (email, Slack, or Teams) so collector outages are caught regardless of remote access method.

07 Golden Image & Provisioning

Build a reusable base image to enable fast, repeatable deployments across sites.

Golden Image Contents

  1. Raspberry Pi OS Lite (64-bit) — Bookworm, fully updated
  2. SSH enabled with authorized keys pre-loaded (password auth disabled)
  3. viyuadmin account with sudo privileges
  4. ufw firewall: allow SSH inbound, allow all outbound
  5. unattended-upgrades installed and configured
  6. Tailscale installed and authenticated to the org Tailnet
  7. Hostname set to auvik-TEMPLATE (placeholder for cloning)
  8. Hardware watchdog timer enabled (/dev/watchdog) for auto-reboot on hangs

Per-Client Customization

After cloning the golden image to a new microSD, customize these before deployment:

SettingAction
HostnameSet to auvik-{clientcode}-{sitecode}
Static IPConfigure static IP or document DHCP reservation
TimezoneSet to client local timezone
Auvik CollectorRun site-specific bash install script from Auvik portal
DocumentationRecord hostname, IP, MAC, serial, Auvik site in documentation platform
Asset LabelPrint and apply physical label to device

Cloning Process

  1. Insert the golden image microSD into a USB reader on a workstation.
  2. Clone using Raspberry Pi Imager (custom image option) or balenaEtcher to the target microSD.
  3. Boot the Pi with the cloned card, SSH in, run per-client customizations.
  4. Run the Auvik bash install script for the target client site.
  5. Verify collector is online in Auvik, then shut down and package for deployment.

08 Deployment Checklist

Use this during each deployment to ensure nothing is missed.

Pre-Deployment (Office)

  • Hardware kit assembled — Pi 5, PoE HAT, case, microSD, patch cable, label
  • Golden image cloned to microSD
  • Per-client config applied — hostname, IP, timezone
  • Auvik collector installed and verified online in portal
  • Tailscale / ZeroTier remote access tested from management network
  • Client SNMP credentials obtained and entered in Auvik
  • Asset label printed and applied to device
  • USB-C backup power supply in tech bag

On-Site Installation

  • Verify client switch port supports 802.3at PoE+ with available power budget
  • Connect Pi to designated switch port via Cat6 patch cable
  • Confirm Pi powers on via PoE — activity LEDs, no separate PSU
  • Verify connectivity — ping gateway, ping 8.8.8.8, curl https://auvik.com
  • Confirm Auvik collector shows online in portal
  • Verify device discovery populating — switches, APs, firewalls appearing
  • Confirm Tailscale/ZeroTier tunnel is active from management network
  • Mount/secure Pi in rack, shelf, or bracket
  • Update client documentation with deployment details
  • Photograph installed Pi for records

Post-Deployment (24–48 Hours)

  • Auvik collector still online and reporting
  • Network topology map populating correctly
  • Auvik offline alert configured for this collector
  • Remote access confirmed working from outside client network

09 Troubleshooting

SymptomLikely CauseResolution
Pi does not power on Switch port is 802.3af-only, or PoE power budget exhausted Verify 802.3at support. Check switch PoE allocation. Test with USB-C 27W PSU to isolate.
Collector offline in Auvik Network issue, Pi crash, SD card failure Check Tailscale device status. Attempt SSH. If unreachable, dispatch tech to verify power/connectivity on-site.
No device discovery SNMP not enabled, wrong credentials, or VLAN isolation Verify SNMP config on target devices. Confirm Pi's IP is in allowed hosts. Check inter-VLAN ACLs.
High CPU / memory usage Site exceeds Pi capacity (>200 devices, >9000 interfaces) Review Auvik device/interface counts. Scale to x86 VM if thresholds are exceeded.
Tailscale not connecting Client firewall blocking outbound to Tailscale control plane Verify outbound HTTPS. Check for SSL inspection. Whitelist Tailscale coordination server domains.
SD card corruption Write wear-out or unexpected power loss Re-flash from golden image. Apply per-client config. For repeat failures, evaluate NVMe upgrade.

10 Ongoing Maintenance

OS Updates

With unattended-upgrades enabled, security patches apply automatically. Quarterly, SSH into each Pi to verify system health, check disk usage (df -h), and review logs. Automate across all deployed Pis via Tailscale SSH.

Hardware Watchdog

The Pi 5 hardware watchdog is configured in the golden image. It auto-reboots the device on system hangs — no post-deployment intervention needed.

SD Card Lifecycle

Plan preventive replacement every 2–3 years. The Auvik collector is not write-heavy, but OS logging and updates accumulate. If a site has repeat SD failures, upgrade to NVMe.

Capacity Thresholds

Monitor in Auvik. If a site approaches 200 devices or 9,000 interfaces, the Pi may not keep up. At that scale, migrate the collector to a dedicated x86 VM.
